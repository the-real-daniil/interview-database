# Общая теория

<details>
<summary><b>Что такое Angular?</b></summary>
 
Angular - это TypeScript-фреймворк с открытым исходным кодом от Google для построения одностраничных (SPA) и масштабируемых клиентских приложений. В отличие от React или Vue, Angular - это не библиотека, а полноценный фреймворк, который "из коробки" предоставляет всё необходимое: систему компонентов, маршрутизацию, формы, HTTP-клиент, инъекцию зависимостей, тестирование и CLI.

Основные особенности:

- **TypeScript** как основной язык разработки

- **Компонентный подход** с использованием декораторов

- **Two-way data binding** через `[(ngModel)]`

- **Dependency Injection** на уровне фреймворка

- **RxJS** для работы с асинхронными потоками данных

- **Angular CLI** для генерации, сборки и тестирования

</details>

<details>
<summary><b>Чем отличается Angular от AngularJS?</b></summary>

- **Язык**: AngularJS написан на JavaScript, Angular (начиная с версии 2) - на TypeScript.

- **Архитектура**: AngularJS основан на контроллерах и `$scope`, Angular - на компонентах и сервисах.

- **Производительность**: Angular использует более эффективный механизм change detection и AOT-компиляцию.

- **Mobile-first**: Angular изначально проектировался с учётом мобильных устройств.

- **Модульность**: Angular полностью модульный (NgModule, standalone-компоненты), что упрощает поддержку больших приложений.

- **Совместимость**: AngularJS и Angular несовместимы между собой, миграция требует переписывания приложения.

</details>

<details>
<summary><b>Что такое NgModule и для чего он нужен?</b></summary>

`NgModule` - это декоратор, который объявляет класс модулем Angular. Модуль группирует логически связанные компоненты, директивы, пайпы и сервисы в единый блок.

Основные свойства декоратора `@NgModule`:

- **`declarations`** - компоненты, директивы и пайпы, принадлежащие модулю

- **`imports`** - другие модули, экспорт которых нужен в текущем модуле

- **`exports`** - то, что модуль предоставляет другим модулям

- **`providers`** - сервисы, доступные через DI

- **`bootstrap`** - корневой компонент приложения (только для AppModule)

Начиная с Angular 14, появились **standalone-компоненты**, которые могут существовать без NgModule, упрощая структуру приложения.

</details>

<details>
<summary><b>Что такое standalone-компоненты?</b></summary>

Standalone-компоненты - это компоненты, директивы и пайпы, которые не нуждаются в объявлении внутри NgModule. Они импортируют свои зависимости напрямую через свойство `imports` декоратора `@Component`.

```TypeScript
@Component({
  selector: 'app-hello',
  standalone: true,
  imports: [CommonModule, FormsModule],
  template: `<h1>Привет, {{ name }}!</h1>`
})
export class HelloComponent {
  name = 'Angular';
}
```

Преимущества:

- Меньше шаблонного кода (не нужно создавать модули)

- Упрощённая структура приложения

- Лучшая tree-shaking оптимизация

- Начиная с Angular 17 - рекомендуемый подход

</details>

# Компоненты

<details>
<summary><b>Что такое компонент в Angular?</b></summary>

Компонент - это базовая единица построения UI в Angular. Каждый компонент состоит из трёх частей:

- **Класс (TypeScript)** - содержит логику и данные

- **Шаблон (HTML)** - описывает разметку

- **Стили (CSS/SCSS)** - описывают внешний вид

Компонент объявляется через декоратор `@Component`:

```TypeScript
@Component({
  selector: 'app-user',
  templateUrl: './user.component.html',
  styleUrls: ['./user.component.scss']
})
export class UserComponent {
  name = 'Иван';
}
```

</details>

<details>
<summary><b>Как организовано взаимодействие между компонентами в Angular?</b></summary>

В Angular есть несколько способов организовать взаимодействие между компонентами:

1. **`@Input()`** - передача данных от родителя к потомку.

    ```TypeScript
    @Input() user: User;
    ```

2. **`@Output()` + `EventEmitter`** - передача событий от потомка к родителю.

    ```TypeScript
    @Output() userChange = new EventEmitter<User>();
    ```

3. **`@ViewChild` / `@ViewChildren`** - получение ссылки на дочерний компонент.

4. **Сервисы** - для передачи данных между несвязанными компонентами через DI (часто с использованием `BehaviorSubject` или `Signal`).

5. **`@ContentChild` / `ng-content`** - проекция контента и доступ к нему.

</details>

# Директивы

<details>
<summary><b>Какие виды директив есть в Angular?</b></summary>

В Angular есть три типа директив:

1. **Компоненты** - директивы со своим шаблоном. Самый частый тип.

2. **Структурные директивы** - изменяют структуру DOM, добавляя или удаляя элементы. Помечаются префиксом `*`.

    - `*ngIf` - условный рендеринг

    - `*ngFor` - рендеринг списков

    - `*ngSwitch` - условный рендеринг с несколькими вариантами

3. **Атрибутивные директивы** - изменяют поведение или внешний вид существующего элемента.

    - `ngClass` - управление CSS-классами

    - `ngStyle` - управление стилями

    - `ngModel` - двустороннее связывание

Также можно создавать собственные директивы через декоратор `@Directive`.

</details>

<details>
<summary><b>Чем отличается <code>*ngIf</code> от <code>[hidden]</code>?</b></summary>

- **`*ngIf`** - структурная директива, которая полностью удаляет элемент из DOM. При повторном появлении компонент пересоздаётся и заново выполняются все хуки жизненного цикла.

- **`[hidden]`** - привязка к нативному атрибуту, которая просто применяет стиль `display: none`. Элемент остаётся в DOM, состояние сохраняется.

**Когда что использовать:**

- `*ngIf` - когда элемент тяжёлый и редко показывается (экономия ресурсов).

- `[hidden]` - когда нужно часто переключать видимость и важна скорость, а не экономия памяти.

</details>

# Data Binding

<details>
<summary><b>Какие виды связывания данных есть в Angular?</b></summary>

В Angular есть четыре основных типа связывания данных (data binding):

1. **Interpolation** - `{{ value }}` - вывод значения из компонента в шаблон.

2. **Property binding** - `[property]="value"` - привязка значения свойства к DOM-свойству элемента.

    ```html
    <img [src]="imageUrl">
    ```

3. **Event binding** - `(event)="handler()"` - подписка на события DOM.

    ```html
    <button (click)="onClick()">Кнопка</button>
    ```

4. **Two-way binding** - `[(ngModel)]="value"` - двустороннее связывание (комбинация property и event binding).

    ```html
    <input [(ngModel)]="name">
    ```

Синтаксис `[(...)]` называется "banana in a box" и под капотом эквивалентен `[value]="name" (valueChange)="name = $event"`.

</details>

# Dependency Injection

<details>
<summary><b>Что такое Dependency Injection в Angular?</b></summary>

**Dependency Injection (DI)** - это паттерн, при котором класс получает свои зависимости извне, а не создаёт их сам. В Angular DI встроен в фреймворк и работает на уровне инжекторов.

Пример инъекции сервиса в компонент:

```TypeScript
@Component({...})
export class UserComponent {
  constructor(private userService: UserService) {}
}
```

Или с помощью функции `inject()` (начиная с Angular 14):

```TypeScript
export class UserComponent {
  private userService = inject(UserService);
}
```

**Преимущества DI:**

- Упрощает тестирование (можно подменить зависимости моками)

- Уменьшает связанность кода

- Позволяет переиспользовать сервисы как синглтоны

- Упрощает управление жизненным циклом зависимостей

</details>

<details>
<summary><b>Что такое сервисы в Angular и зачем они нужны?</b></summary>

**Сервис** - это класс, помеченный декоратором `@Injectable`, который предоставляет общую функциональность нескольким компонентам. Сервисы используются для:

- Работы с HTTP-запросами

- Хранения общего состояния приложения

- Логирования, аутентификации, валидации

- Бизнес-логики, не связанной с UI

```TypeScript
@Injectable({ providedIn: 'root' })
export class UserService {
  getUsers() {
    return this.http.get('/api/users');
  }
}
```

Свойство `providedIn: 'root'` делает сервис синглтоном на уровне всего приложения и обеспечивает tree-shaking, если сервис нигде не используется.

</details>

# Жизненный цикл

<details>
<summary><b>Какие хуки жизненного цикла есть у компонента Angular?</b></summary>

Основные хуки жизненного цикла компонента:

1. **`ngOnChanges`** - вызывается при изменении значения `@Input()` свойств. Принимает объект `SimpleChanges`.

2. **`ngOnInit`** - вызывается один раз после первого `ngOnChanges`. Используется для инициализации компонента.

3. **`ngDoCheck`** - вызывается при каждом цикле change detection. Используется для пользовательской логики обнаружения изменений.

4. **`ngAfterContentInit`** - вызывается после проекции контента через `<ng-content>` в компонент.

5. **`ngAfterContentChecked`** - вызывается после проверки спроецированного контента.

6. **`ngAfterViewInit`** - вызывается после инициализации представления компонента и его дочерних представлений.

7. **`ngAfterViewChecked`** - вызывается после проверки представления.

8. **`ngOnDestroy`** - вызывается перед уничтожением компонента. Используется для очистки подписок и таймеров.

</details>

<details>
<summary><b>Чем отличаются <code>ngOnInit</code> и конструктор компонента?</b></summary>

- **Конструктор** вызывается при создании экземпляра класса. На этом этапе ещё не доступны `@Input()` свойства, и Angular ещё не закончил инициализацию компонента. Конструктор должен использоваться только для инъекции зависимостей.

- **`ngOnInit`** вызывается после того, как Angular инициализировал все `@Input()` свойства и применил начальные значения. Это правильное место для:

    - Загрузки данных с сервера

    - Инициализации компонента на основе входных данных

    - Подписок на observables

</details>

# Change Detection

<details>
<summary><b>Как работает Change Detection в Angular?</b></summary>

**Change Detection (CD)** - это механизм, с помощью которого Angular отслеживает изменения данных и обновляет DOM.

Как работает:

1. При каждом асинхронном событии (клик, HTTP-ответ, таймер) Angular запускает цикл CD.

2. Цикл начинается с корневого компонента и идёт вниз по дереву компонентов.

3. Для каждого компонента Angular сравнивает текущие значения с предыдущими.

4. Если значения изменились - обновляется DOM.

Angular использует библиотеку **Zone.js**, которая патчит асинхронные API браузера и сообщает Angular о необходимости запуска CD.

**Стратегии Change Detection:**

- **Default** - проверяет всё дерево компонентов при каждом событии.

- **OnPush** - проверяет компонент только если:

    - Изменилась ссылка на `@Input()` свойство

    - Произошло событие внутри компонента

    - Вручную вызван `markForCheck()` или `detectChanges()`

    - Сработал async pipe

</details>

<details>
<summary><b>Что такое стратегия OnPush и когда её использовать?</b></summary>

`ChangeDetectionStrategy.OnPush` - стратегия обнаружения изменений, при которой компонент проверяется не на каждое событие, а только при определённых условиях:

- Изменилась **ссылка** на `@Input()` (для объектов важно создавать новые объекты, а не мутировать)

- Событие произошло в самом компоненте или его потомках

- В шаблоне сработал `async` pipe

- Вручную вызвано `ChangeDetectorRef.markForCheck()` или `detectChanges()`

```TypeScript
@Component({
  selector: 'app-user',
  changeDetection: ChangeDetectionStrategy.OnPush,
  template: '...'
})
export class UserComponent {}
```

**Когда использовать:**

- Для оптимизации производительности в больших приложениях

- Для компонентов с большим количеством данных

- Когда работаем с иммутабельными данными или Observable + async pipe

</details>

# RxJS и работа с асинхронностью

<details>
<summary><b>Что такое Observable и чем он отличается от Promise?</b></summary>

**Observable** - это объект из библиотеки RxJS, представляющий поток значений во времени. Angular активно использует Observable для работы с HTTP-запросами, событиями форм, маршрутизацией и т.д.

**Отличия Observable от Promise:**

| Promise | Observable |
| --- | --- |
| Возвращает одно значение | Возвращает поток значений (0..N) |
| Eager (выполняется сразу) | Lazy (выполняется при подписке) |
| Нельзя отменить | Можно отписаться через `unsubscribe()` |
| Нет операторов | Богатая библиотека операторов (map, filter, switchMap и т.д.) |
| Только async | Может быть как sync, так и async |

```TypeScript
this.userService.getUsers()
  .pipe(
    map(users => users.filter(u => u.active)),
    take(1)
  )
  .subscribe(users => console.log(users));
```

</details>

<details>
<summary><b>Что такое async pipe и в чём его преимущества?</b></summary>

`async` - это встроенный pipe в Angular, который автоматически подписывается на `Observable` или `Promise` и возвращает последнее эмитированное значение. При уничтожении компонента он автоматически отписывается, предотвращая утечки памяти.

```html
<div *ngIf="user$ | async as user">
  {{ user.name }}
</div>
```

**Преимущества async pipe:**

- Автоматическая отписка при уничтожении компонента

- Меньше шаблонного кода (нет нужды вручную делать `.subscribe()`)

- При использовании со стратегией OnPush автоматически вызывает `markForCheck()`

- Упрощает работу с реактивными потоками данных

</details>

# Формы

<details>
<summary><b>Чем отличаются Template-driven Forms от Reactive Forms?</b></summary>

В Angular есть два подхода к работе с формами:

**Template-driven Forms:**

- Логика формы описывается в шаблоне через директивы `ngModel`, `ngForm`

- Подходит для простых форм

- Меньше кода в компоненте

- Сложнее тестировать

- Асинхронные по природе

```html
<form #f="ngForm" (ngSubmit)="submit(f.value)">
  <input name="email" ngModel required>
</form>
```

**Reactive Forms:**

- Логика формы описывается в TypeScript через `FormControl`, `FormGroup`, `FormBuilder`

- Подходит для сложных динамических форм

- Лучше типизируется и тестируется

- Синхронные по природе

- Иммутабельность данных

```TypeScript
form = new FormGroup({
  email: new FormControl('', [Validators.required, Validators.email])
});
```

Reactive Forms - предпочтительный подход для большинства случаев.

</details>

# Маршрутизация

<details>
<summary><b>Что такое Router и как он работает в Angular?</b></summary>

**Angular Router** - встроенный механизм маршрутизации для SPA. Он сопоставляет URL с компонентами и управляет навигацией без перезагрузки страницы.

Основные понятия:

- **Routes** - массив объектов, описывающих маршруты приложения.

- **`<router-outlet>`** - директива-плейсхолдер, в которую рендерится компонент текущего маршрута.

- **`routerLink`** - директива для навигации между маршрутами.

- **`ActivatedRoute`** - сервис для доступа к параметрам текущего маршрута.

- **`Router`** - сервис для программной навигации.

```TypeScript
const routes: Routes = [
  { path: '', component: HomeComponent },
  { path: 'users/:id', component: UserComponent },
  { path: '**', component: NotFoundComponent }
];
```

**Дополнительные возможности:**

- **Lazy Loading** - подгрузка модулей по требованию через `loadChildren`

- **Guards** - защита маршрутов (`CanActivate`, `CanDeactivate`)

- **Resolvers** - предзагрузка данных перед активацией маршрута

</details>

# Pipes

<details>
<summary><b>Что такое Pipe и какие бывают виды?</b></summary>

**Pipe** - это функция-преобразователь данных в шаблоне Angular. Используется через символ `|`.

```html
{{ price | currency:'RUB' }}
{{ date | date:'dd.MM.yyyy' }}
{{ users | async }}
```

**Виды pipes:**

1. **Pure pipe (по умолчанию)** - вызывается только при изменении ссылки на входное значение. Это позволяет Angular эффективно кэшировать результат.

2. **Impure pipe** - вызывается при каждом цикле change detection. Используется для работы с мутируемыми данными, но негативно влияет на производительность.

```TypeScript
@Pipe({ name: 'myPipe', pure: false })
```

Встроенные pipes: `DatePipe`, `CurrencyPipe`, `UpperCasePipe`, `LowerCasePipe`, `JsonPipe`, `AsyncPipe`, `DecimalPipe` и др.

</details>

# Signals

<details>
<summary><b>Что такое Signals в Angular?</b></summary>

**Signals** - это новый механизм реактивности, появившийся в Angular 16. Signal - это обёртка над значением, которая уведомляет заинтересованные части приложения об изменении этого значения.

```TypeScript
import { signal, computed, effect } from '@angular/core';

const count = signal(0);
const doubled = computed(() => count() * 2);

effect(() => {
  console.log('Count:', count());
});

count.set(5);
count.update(c => c + 1);
```

**Основные функции:**

- **`signal()`** - создаёт реактивное значение

- **`computed()`** - создаёт вычисляемое значение на основе других signals

- **`effect()`** - выполняет побочные эффекты при изменении signals

**Преимущества:**

- Точная и предсказуемая реактивность (без Zone.js)

- Лучшая производительность за счёт точечных обновлений

- Более простая модель ментально, чем RxJS, для локального состояния

- Возможность будущего отказа от Zone.js (zoneless Angular)

</details>
