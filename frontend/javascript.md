На текущий момент является одним из самых популярных языков программирования в мире и может запускаться не только в браузере, но и в других средах, например, на сервере, используя Node JS.

### Ответ для лунтиков

Представьте веб-страницу как дом:

- **HTML** — это каркас и стены дома, он задаёт структуру.

- **CSS** — это краска и декор, он делает дом красивым.

- **JavaScript** — это электричество и бытовые приборы, которые оживляют дом. Благодаря JavaScript в доме загорается свет при нажатии на выключатель, работает телевизор и открываются автоматические двери.

Без JavaScript ваш сайт был бы как красивый, но не функционирующий дом. С JavaScript он становится "живым" и реагирует на действия посетителей: кнопки нажимаются, формы отправляются, элементы на странице могут изменяться без перезагрузки. Он позволяет создавать динамичный и удобный для пользователя опыт на веб-сайте.

<summary><b>Что означают атрибуты async и defer?</b></summary>

Браузер загружает и отображает HTML постепенно. Особенно это заметно при медленном интернет-соединении: браузер не ждёт, пока страница загрузится целиком, а показывает ту часть, которую успел загрузить.

Если браузер видит тег `<script>`, то он по стандарту обязан сначала выполнить его, а потом показать оставшуюся часть страницы. Т.е. загрузка получается синхронной сверху вниз.

Если скрипт – внешний, то пока браузер не выполнит его, он не покажет часть страницы под ним.

При указании атрибута `async` и `defer` браузер начинает подгружать скрипты в асинхронном режиме и не тормозит отображение html.

Но `defer` используют в том случае, когда важен порядок загрузки скриптов, скрипты будут загружаться асинхронно, но последовательно.

И второе отличие, `defer` запустит скрипт только тогда, когда весь html будет обработан браузером и дерево DOM готово (но до события `DOMContentLoaded`), а `async` начинает выполнять скрипт сразу после его загрузки. Поэтому важно, если скрипт работает с html-элементами использовать именно `defer`.

<summary><b>Какие способы объявления переменных вы знаете? Чем они отличаются?</b></summary>

Объявить переменную можно 4 способами

1. `var a = 5;` или `a = 5;`

    - область видимости: глобальная или функциональная (если объявлена внутри функции, то функциональная, если нет — то глобальная)

    - происходит hoisting (всплытие) до верхней границы области видимости, поэтому внутри функции к переменной можно обращаться до ее фактического объявления

    - при объявлении в глобальной области видимости, переменная записывается в window

    - устаревший способ объявления переменных

2. `let a = 5;`

    - область видимости: блочная

    - тоже происходит hoisting (всплытие), но обращаться к переменным до их объявления нельзя, будет ошибка (дополнительно можно почитать про temporal dead zone)

    - переменной можно переприсваивать значения

3. `const a = 5;`

    - область видимости: блочная

    - тоже происходит hoisting (всплытие), но обращаться к переменным до их объявления нельзя, будет ошибка (дополнительно можно почитать про temporal dead zone)

    - переменной НЕЛЬЗЯ переприсвоить значение (константа)

<summary><b>Что такое строгий режим? Для чего он нужен?</b></summary>

Строгий режим — это более строгий режим работы javascript кода, который позволяет движкам JavaScript выполнять некоторые оптимизации, а также выбрасывает дополнительные исключения в некоторых местах кода, которые в нестрогом режиме допускаются. Например, в нестрогом режиме при присвоении значения несуществующей переменной создается новая глобальная переменная. В строгом режиме выбрасывается ошибка `ReferenceError`.

Еще один частый пример различного поведения в строгом и нестрогом режиме: значение `this` в функции, вызванной без объекта, в строгом режиме равняется `undefined`, а в нестрогом - `global` (в Node JS) или `window` (в браузере).

Необходимость в использовании строгого режима появилась в момент, когда некоторые принципы работы JavaScript были признаны устаревшими. При этом, чтобы сохранить совместимость старых и новых версий программ разработчики языка приняли решения добавить директиву `'use strict'`, которая включает строгий режим в конкретном файле или функции.

Сейчас большинство современных программ на JavaScript пишутся в строгом режиме.

<summary><b>Что такое тернарный оператор?</b></summary>

Тернарный оператор — это упрощённая форма условного оператора, который позволяет выполнять выбор между двумя выражениями в зависимости от истинности заданного условия. Обычно записывается как условие ? выражение1 : выражение2.

Структура тернарного оператора:

- Условие: Любое выражение, которое оценивается как true (истина) или false (ложь).

- Выражение1: Выполняется, если условие истинно (true).

- Выражение2: Выполняется, если условие ложно (false).

Пример:

```JavaScript
let age = 18;
let status = age >= 18 ? 'взрослый' : 'несовершеннолетний';
console.log(status); // Выведет: взрослый
```

В этом примере условие age >= 18 проверяет, является ли возраст человека 18 лет или более. Если условие истинно, переменной status присваивается значение 'взрослый'. В противном случае — 'несовершеннолетний'.

<summary><b>Как работают логические операторы <code>&&</code> и <code>||</code>?</b></summary>

Оператор `&&` выполняет следующие действия:

- Вычисляет операнды слева направо.

- Каждый операнд преобразует в логическое значение. Если результат `false`, останавливается и возвращает исходное значение этого операнда.

- Если все операнды были истинными, возвращается последний.

Оператор `||` выполняет следующие действия:

- Вычисляет операнды слева направо.

- Каждый операнд конвертирует в логическое значение. Если результат `true`, останавливается и возвращает исходное значение этого операнда.

- Если все операнды являются ложными (`false`), возвращает последний из них.

<summary><b>Как работает оператор нулевого слияния <code>??</code>?</b></summary>

Оператор `??` выполняет следующие действия:

- Вычисляет операнды слева направо.

- Если операнд не равен `null` или `undefined`, останавливается и возвращает значение этого операнда.

- Если все операнды равны `null` или `undefined`, возвращается последний.

Оператор нулевого слияния появился недавно и в некоторых случаях стал заменой оператору `||`, когда не требуется приведение к типу boolean.

<details>
<summary><b>Как сравнить два объекта по значению?</b></summary>

В отличие от примитивов (например, чисел, строк), объекты сравниваются по ссылке, и даже если два объекта имеют одинаковые свойства и значения, они будут считаться разными, если ссылки на них различны.

Для сравнения объектов по значению можно использовать:

1. Рекурсивное сравнение объектов (глубокое сравнение)

    ```JavaScript
    function deepEqual(obj1, obj2) {
        // Проверяем, равны ли типы
        if (typeof obj1 !== typeof obj2) return false;

        // Если объекты равны по ссылке, они равны
        if (obj1 === obj2) return true;

        // Если объекты не являются объектами, значит, это примитивы
        if (typeof obj1 !== 'object' || obj1 === null || obj2 === null) return false;

        // Получаем ключи объектов
        const keys1 = Object.keys(obj1);
        const keys2 = Object.keys(obj2);

        // Если количество ключей разное, объекты не равны
        if (keys1.length !== keys2.length) return false;

        // Сравниваем значения всех ключей
        for (let key of keys1) {
            // Проверяем, есть ли ключ во втором объекте и рекурсивно сравниваем их значения
            if (!keys2.includes(key) || !deepEqual(obj1[key], obj2[key])) {
                return false;
            }
        }

        return true;
    }
    ```

2. Для простых объектов можно использовать метод `JSON.stringify` для сравнения их в виде строк

    ```JavaScript
    function simpleEqual(obj1, obj2) {
        return JSON.stringify(obj1) === JSON.stringify(obj2);
    }
    ```

<summary><b>Какие методы массивов вы знаете?</b></summary>

Чаще всего используются следующие методы массивов:

- Метод `forEach` ничего не возвращает, его используют только для перебора, как более «элегантный» вариант, чем обычный цикл `for`.

- Метод `includes(value[, fromIndex])` – возвращает `true`, если в массиве имеется элемент `value`, в противном случае `false`.

<details>
<summary><b>Как работает сборщик мусора в JavaScript?</b></summary>

Конкретная реализация сборщика мусора зависит от движка, на котором запускается JavaScript. Но есть некоторые общие принципы работы сборщика мусора:

- Работает в отдельном потоке

- Использует принцип достижимости. Если значение недостижимо, то оно удаляется из памяти. Например, если в памяти остался объект, на который нет ссылок, то он удаляется

- Использует «алгоритм пометок» (от англ. «mark-and-sweep»). Согласно этому алгоритму, сборщик мусора регулярно выполняет следующие шаги:

    - Сборщик мусора «помечает» (запоминает) все корневые объекты.

    - Затем он идёт по ним и «помечает» все ссылки из них.

    - Затем он идёт по отмеченным объектам и отмечает их ссылки. Все посещённые объекты запоминаются, чтобы в будущем не посещать один и тот же объект дважды.

    - …И так далее, пока не будут посещены все достижимые (из корней) ссылки.

    - Все непомеченные объекты удаляются.

- Использует различные приемы оптимизации, например: сборка по поколениям (Generational collection), инкрементальная сборка (Incremental collection), сборка в свободное время (Idle-time collection)

<summary><b>Чем WeakMap и WeakSet отличаются от Map и Set?</b></summary>

1. В качестве ключей в WeakMap и значений в WeakSet могут использоваться только объекты (не примитивные значения)

<summary><b>Почему в javascript можно вызвать метод у примитива (например, <code>str.toLowerCase</code>)?</b></summary>

Каждый примитив имеет свой собственный «объект-обёртку», которые называются: `String`, `Number`, `Boolean`, `Symbol` и `BigInt`. Таким образом, они имеют разный набор методов.

К примеру, существует метод [str.toUpperCase()](https://developer.mozilla.org/ru/docs/Web/JavaScript/Reference/Global_Objects/String/toUpperCase), который возвращает строку в верхнем регистре.

Вот, как он работает:

`let str = "Привет";`

`alert( str.toUpperCase() ); // ПРИВЕТ`

Вот, что на самом деле происходит в `str.toUpperCase()`:

1. Строка `str` – примитив. В момент обращения к его свойству, создаётся специальный объект класса `String`, который cодержит значение строки и имеет такие методы, как `toUpperCase()`, `toLowerCase()` и прочие методы `String`.

2. Этот метод запускается и возвращает новую строку (показывается в `alert`).

3. Специальный объект удаляется, оставляя только примитив `str`.

<summary><b>Можно ли использовать javascript для точных вычислений?</b></summary>

Скорее нет, так как у JavaScript есть проблемы с точностью вычислений для чисел в плавающей точкой. Это происходит из-за того, что внутри JavaScript число представлено в виде 64-битного формата [IEEE-754](https://ru.wikipedia.org/wiki/IEEE_754-1985). Для хранения числа используется 64 бита: 52 из них используется для хранения цифр, 11 для хранения положения десятичной точки и один бит отведён на хранение знака.

Таким образом нет возможности для хранения точных значений 0.1 или 0.2, используя двоичную систему, точно также, как нет возможности хранить одну третью в десятичной системе счисления.

Числовой формат IEEE-754 решает эту проблему путём округления до ближайшего возможного числа. Правила округления обычно не позволяют нам увидеть эту «крошечную потерю точности», но она существует.

Наиболее надёжный способ обойти это — округлить результат используя метод [toFixed(n)](https://developer.mozilla.org/ru/docs/Web/JavaScript/Reference/Global_Objects/Number/toFixed):

# Асинхронность

<details>
<summary><b>Чем отличается асинхронность от параллельности? JavaScript асинхронный или параллельный?</b></summary>

Асинхронность относится к выполнению задач таким образом, что они не блокируют основной поток выполнения программы. В JavaScript асинхронность часто используется для работы с операциями, которые могут занять какое-то время, например, запросами к серверу или чтением файлов. Когда ты выполняешь асинхронную операцию, ты можешь продолжать выполнять другие задачи, пока эта операция завершается.

Параллельность же означает выполнение нескольких задач одновременно. Это более низкоуровневый концепт, который связан с наличием нескольких потоков или процессов. Например, в языках программирования с поддержкой потоков, таких как Java или C++, параллельность может быть реализована с помощью создания нескольких потоков, которые выполняются одновременно на разных ядрах процессора.

В JavaScript основной поток выполнения кода однопоточный, что означает, что он выполняет задачи последовательно. Тем не менее, JavaScript может использовать Web Workers для выполнения задач параллельно, но это отличается от асинхронного выполнения. Web Workers позволяют выполнять код в фоновом потоке, не блокируя основной поток, но это отдельные потоки, которые могут выполнять задачи параллельно.

</details>

<details>
<summary><b>Что такое Promise?</b></summary>

Promise – это специальный объект, предназначенный для работы с асинхронным кодом и который содержит своё состояние.

Вначале `pending` («ожидание»), затем – одно из: `fulfilled` («выполнено успешно») или `rejected` («выполнено с ошибкой»).

При создании объекта промиса в конструктор передается функция-колбэк.

Её аргументы `resolve` и `reject` – это функции, которые поставляются самим языком JavaScript.

Когда промис получает результат, он должен вызвать один из этих колбэков:

- `resolve(value)` — если работа завершилась успешно, с результатом `value`.

- `reject(error)` — если произошла ошибка, `error` – объект ошибки.

Для обработки результатов промиса используются методы:

- `then` — принимает колбэк для обработки результата, вызывается только в случае успеха (`resolve(value)`)

- `catch` — принимает колбэк для обработки ошибки, вызывается только в случае ошибки (`reject(error)`)

- `finally` — принимает колбэк для обработки любого результата, вызывается в любом случае, независимо от успеха промиса

<details>
<summary><b>Какие способы работы с асинхронным кодом есть в javascript?</b></summary>

1. Колбэки (`callback`)

    В функцию, выполняющую асинхронную операцию, передается колбэк, который вызывается при завершении операции

2. Промисы (`Promise`)

    Асинхронная функция возвращает промис, результат выполнения которого может быть обработан внешним кодом.

    Существует несколько способов обработки результата промиса:

    1. Обработчики `then`, `catch`, `finally`

    2. Ключевые слова `async`, `await`

3. Функции-генераторы

    Генераторы позволяют приостанавливать выполнение функции и возобновлять его позже. В сочетании с `Promise`, они могут использоваться для асинхронных операций.

    ```JavaScript
    function* fetchData() {
        let data = yield new Promise((resolve) => {
            setTimeout(() => resolve("Data loaded"), 1000);
        });
        console.log(data); // "Data loaded"
    }

    const gen = fetchData();
    gen.next().value.then((data) => gen.next(data));
    ```

<summary><b>Что такое Event Loop?</b></summary>

Event Loop - это механизм управления очередностью выполнения кода, который реализован в движках для запуска JavaScript (например, V8 в Chrome). В первую очередь необходим для реализации асинхронности.

Включает в себя:

- Call Stack – стек вызовов. В него попадают все функции, которые необходимо исполнить

- Macrotask Queue – очередь макрозадач. В нее попадают колбэки `setTimeout`, `setInterval`, обработчики пользовательских событий, задача выполнения скрипта после его подгрузки (тег `<script>`)

- Microtask Queue – очередь микрозадач. В нее автоматически попадают обработчики промисов. Также можно воспользоваться встроенным методом `queueMicrotask`, чтобы вручную поместить функцию в очередь микрозадач.

    Очередь микрозадач имеет более высокие приоритет, чем очередь макрозадач, поэтому выполняется в первую очередь.

    Браузер не рендерит изменения, пока все задачи из очередь микрозадач не будут выполнены.

- Render callbacks — очередь рендер-колбэков, которые выполняются строго перед отрисовкой страницы. Добавить колбэки в эту очередь можно через `requestAnimationFrame`

- Render Queue (только в браузере) – очередь задач по отрисовке страницы

- Idle callbacks — очередь idle колбэков, которые выполняются, если после завершения всего цикла у браузера остается время до следующего кадра (браузер стремится к 60 fps)

Event Loop работает по следующему алгоритму:

1. Выполнить все синхронные задачи из Call Stack

2. Исполнить все *микрозадачи*:

    - Пока очередь микрозадач не пуста: - Выбрать из очереди и исполнить старейшую микрозадачу

3. Выбрать и исполнить старейшую задачу из очереди *макрозадач* (например, «script»).

4. Выполнить колбэки из очереди Render callbacks

5. Отрисовать изменения страницы, если они есть.

6. Если осталось время до следующего кадра, то выполнить idle колбэки и ждать появления новых задач

7. Перейти к шагу 1.

<summary><b>Как использовать async/await для работы с промисами?</b></summary>

Существует специальный синтаксис для работы с промисами `async`/`await`.

Функция, обозначенная как `async` всегда возвращает `Promise`.

# Функции

<details>
<summary><b>function expression и function declaration. Что это такое и в чем разница?</b></summary>

Function declaration – привычный способ объявить функцию. Например:

```JavaScript
function sayHi() {
  console.log('Hi');
}
```

Существует ещё один синтаксис создания функций, который называется Function Expression

```JavaScript
const sayHi = function() {
  console.log('Hi');
}
```

В коде выше функция создаётся и явно присваивается переменной, как любое другое значение. По сути без разницы, как мы определили функцию, это просто значение, хранимое в переменной sayHi.

Отличия:

- Function Declaration обрабатываются перед выполнением блока кода. Они видны во всём блоке.

- Функции, объявленные при помощи Function Expression, создаются, только когда поток выполнения достигает их.

```JavaScript
const parentheses = (str = '') => {

		return true
	}
	
	let beginnersCount = 0
	let endingsCount = 0
	let i = 0
	
	while (i < str.length) {
	    if (str[i] === '{') {
	        ++beginnersCount
	    }
	
	    if (str[i] === '}') {

	    }

	    }

function parentheses(str) {

    }
```

<summary><b>Реализовать функцию, вычисляющую выражения в обратной польской нотации (Sber Devices)</b></summary>

### Ответ

```JavaScript
const calc = (expr = '') => {
	if (expr.length === 0) {
		return 0
	}

	let result = 0
	let operands = []
	
	const arr = expr.split(' ')
	
	for (let i = 0; i < arr.length; ++i) {
	    const elem = arr[i]
	
	    switch (elem) {
	        case '+':
	            if (operands.length === 1) {
	                throw new Error('Error in Syntax')
	            }
	            operands.push(operands.pop() + operands.pop())
	
	            break
	        case '*':
	            if (operands.length === 1) {
	                throw new Error('Error in Syntax')
	            }
	            operands.push(operands.pop() * operands.pop())
	            break
	        case '-':
	            if (operands.length === 1) {
	                throw new Error('Error in Syntax')
	            }
	            operands.push(operands.pop() - operands.pop())
	            break
	        default:
	            if (Number.isNan(+arr[i])) {
	                throw new Error('Error in Operands')
	            }
	            operands.push(arr[i])
	    }
	}
	
	if (operands.length > 1) {
	    throw new Error('Error in Syntax')
	}
	
	return operands[0]
}
```

```JavaScript
	})
```

### Ответ

```JavaScript
class Promise {
	queue = []
	
	constructor(cb) {
	    cb(this.#resolve.bind(this))
	}
	
	#resolve(value) {
	    let result = value
	
	    queue.forEach(cb => {
	        result = cb(result)
	    })
	}
	
	then(cb) {
	    queue.push(cb)
	
	    return this
	}
}
```

<summary><b>Функция fetchRetry с повторными попытками (Сбер, HR-платформа Пульс)</b></summary>

Необходимо написать функцию, которая на вход принимает URL и асинхронно ходит по этому `url` GET запросом и возвращает данные. Если во время запроса произошла ошибка, то пробовать запросить ещё и так 5 раз. Если в итоге информацию получить не удалось, вернуть ошибку "URL недоступен".

```JavaScript
function fetchRetry() {}
```

### Ответ

```JavaScript
async function fetchRetry(url, retries = 5) {
    for (let i = 0; i < retries; i++) {
        try {
            const response = await fetch(url);

            if (!response.ok) {
                throw new Error(`Request failed with status ${response.status}`);
            }

            const data = await response.json();

            return data;
        } catch (error) {
            if (i === retries - 1) {
                throw new Error("URL недоступен");
            }
        }
    }
}
```

```JavaScript
function debounce(fn, delay) {
  let timer = null
  
  return function(...args) {
    if (timer) {
      clearTimeout(timer)
    }
    
    timer = setTimeout(() => {
      fn(...args)
    }, delay)
  }
}

const debouncedClg = debounce(clg, 3000)
debouncedClg(1) // не выведется
```
