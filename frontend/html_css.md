<details>
<summary><b>Специфичность селекторов. Что это и как работает?</b></summary>

Каждый селектор в CSS имеет свой "вес" или степень специфичности. Этот вес определяет, насколько конкретно селектор нацелен на элемент. Чем более специфичен селектор, тем выше его приоритет при применении стилей.

Специфичность определяется по следующей схеме:

1. **Inline-стили**: стили, добавленные непосредственно в элемент через атрибут `style=""`, имеют наивысшую специфичность.

2. **Селекторы по ID**: селекторы, использующие идентификатор элемента (`#id`), имеют высокий уровень специфичности.

3. **Классы, атрибуты и псевдоклассы**: селекторы, использующие классы (`.class`), атрибуты (`[type="text"]`) и псевдоклассы (`:hover`), имеют средний уровень специфичности.

4. **Элементные селекторы и псевдоэлементы**: селекторы, использующие имена тегов (`div`, `p`) и псевдоэлементы (`::before`), имеют низкий уровень специфичности.

Особое место занимает директива `!important`. Она не зависит от остальных правил специфичности и имеет наивысший приоритет.

БЭМ (Блок, Элемент, Модификатор) — это методология для организации кода в HTML и CSS, которая помогает делать код более структурированным и удобным для поддержки.

**Модификатор** — состояние или внешний вид блока или элемента, например, большая кнопка (`.button--large`).

<summary><b>Что такое свойство box-sizing и какие значения оно принимает?</b></summary>

`box-sizing` — это свойство в CSS, которое определяет, как рассчитывается ширина и высота элемента.

Оно имеет два значения:

- **content-box** (по умолчанию): ширина и высота учитывают только содержимое. Если добавить `border` или отступы, размер элемента увеличится.

- **border-box**: ширина и высота включают границы и отступы, так что общий размер элемента остаётся таким, как задано, независимо от добавленных границ и отступов.

Обычно используют `box-sizing: border-box;`, чтобы размеры элемента были предсказуемыми и не менялись из-за добавленных отступов и рамок.

<summary><b>Какие есть способы скрыть html-элемент с помощью CSS и в чем их различия?</b></summary>

1. **`display: none;`** — элемент полностью удаляется из потока страницы, и соседние элементы ведут себя так, будто его нет.

2. **`visibility: hidden;`** — элемент остаётся на месте, занимает место на странице, но становится невидимым.

3. **`opacity: 0;`** — элемент остаётся видимым для браузера и занимает место, но становится полностью прозрачным. Можно анимировать, чтобы плавно скрыть элемент.

4. **`position: absolute; top: -9999px;`** — элемент перемещается за пределы видимой области, его нет на экране, но он по-прежнему существует в DOM и доступен для скринридеров.

Есть еще более извращенные способы, типа использование `clip-path`, но вышеизложенных способов обычно достаточно для решения любых задач.

<summary><b>Какие есть значения у css-свойства position?</b></summary>

CSS-свойство `position` управляет тем, как элемент располагается на странице. Его основные значения:

1. **static** (по умолчанию): элемент следует обычному потоку страницы, без специального позиционирования.

2. **relative**: элемент остаётся в потоке, но можно смещать его относительно исходного положения с помощью `top`, `right`, `bottom`, `left`.

3. **absolute**: элемент удаляется из потока и позиционируется относительно ближайшего родителя с `position: relative`, `absolute`, или `fixed`. Если таких нет, позиционируется относительно `body`.

4. **fixed**: элемент удаляется из потока и остаётся закреплённым в одном месте экрана, даже при прокрутке страницы. Позиционируется относительно окна браузера.

5. **sticky**: элемент ведёт себя как `relative`, пока не достигает заданной позиции при прокрутке, затем «прилипает» и остаётся на месте.

<summary><b>Какие есть способы центрировать элемент с помощью css?</b></summary>

В CSS есть несколько способов центрировать элемент, в зависимости от того, что именно нужно центрировать — текст, блок, или элемент внутри родителя:

1. **Центрирование текста по горизонтали**: можно использовать `text-align: center;` на родителе для текста внутри него.

2. **Центрирование блока по горизонтали**: для блока с фиксированной шириной можно использовать `margin: 0 auto;`, если родитель имеет `display: block` и заданную ширину.

3. **Абсолютное центрирование**: при `position: absolute;` или `fixed;` на элементе и `top: 50%; left: 50%;` можно добавить `transform: translate(-50%, -50%);`, чтобы сдвинуть элемент обратно к центру.

4. **Центрирование с Flexbox**: можно задать родителю `display: flex; justify-content: center; align-items: center;` для центрирования по горизонтали и вертикали одновременно.

5. **Центрирование с Grid**: можно установить на родителе `display: grid; place-items: center;` — это центрирует элемент и по горизонтали, и по вертикали.

<summary><b>За что отвечает свойство z-index?</b></summary>

Свойство **`z-index`** определяет порядок наложения элементов на странице, то есть какой элемент будет находиться выше других, если они перекрываются. Чем больше значение `z-index`, тем выше элемент располагается.

- **Положительные значения** поднимают элемент над теми, у кого значение ниже.

- **Отрицательные значения** опускают элемент под теми, у кого значение выше или равно нулю.

У свойства z-index есть некоторые особенности работы, о которых следует помнить:

1. **Работает только на позиционированных элементах**: `z-index` действует только на элементы с `position: relative`, `absolute`, `fixed`, или `sticky`.

    ```CSS
    .box1 {
      position: relative;
      z-index: 10; /* Этот элемент может перекрывать другие */
    }
    .box2 {
      position: static; /* z-index не действует */
    }
    ```

1. Если задать элементу `z-index`, он создает "свой слой" или другими словами контекст наложения (см. дополнительные материалы). Внутри этого слоя элементы будут располагаться относительно друг друга, но не смогут выйти за его пределы.

    ```HTML
    <div class="parent" style="position: relative; z-index: 10;">
      <div class="child" style="position: absolute; z-index: 100;">
        Дочерний элемент
      </div>
    </div>
    <div class="outside" style="position: relative; z-index: 20;">
      Вне родителя
    </div>
    ```

    Здесь `outside` окажется поверх `parent`, несмотря на высокий `z-index` у `child`, так как `child` не может выйти за пределы `parent`.

<details>
<summary><b>Чем отличаются блочные элементы от строчных?</b></summary>

Блочные и строчные элементы в HTML различаются тем, как они отображаются на странице и как ведут себя в макете.

**Блочные элементы:**

- Занимают всю ширину родителя по умолчанию.

- Начинаются с новой строки, "раздвигая" элементы ниже.

- Можно задавать размеры (ширину и высоту).

- Примеры: `<div>`, `<p>`, `<h1>`, `<section>`, `<article>`.

**Строчные элементы:**

- Занимают только столько места, сколько требуется их содержимому.

- Не начинаются с новой строки, размещаясь на одной строке с другими строчными элементами.

- Нельзя задать ширину и высоту (в CSS размер задаётся, но работает не всегда).

- Примеры: `<span>`, `<a>`, `<strong>`, `<em>`.

**Пример**: Если в HTML есть `<div>` и `<span>`, `<div>` будет занимать всю строку, а `<span>` — только вокруг текста, оставаясь на одной строке с другими элементами.

<details>
<summary><b>Как сделать интерфейс сайта адаптивным под разные размеры экрана?</b></summary>

Чтобы сделать интерфейс сайта адаптивным, можно использовать несколько ключевых техник:

1. **Медиа-запросы**: позволяют менять стили в зависимости от ширины устройства. Например:

    ```CSS
    @media (max-width: 768px) {
      .container {
        flex-direction: column;
      }
    }
    ```

    Здесь для экранов до 768px шириной элементы в `.container` будут выстраиваться в колонку.

1. **Процентные значения и единицы vw/vh**: используйте проценты, `vw` (ширина экрана) и `vh` (высота экрана), чтобы элементы могли изменять размер в зависимости от размеров окна браузера.

    ```CSS
    .box {
      width: 50%; /* Ширина будет 50% от родителя */
    }
    ```

1. **Flexbox и Grid** помогают гибко управлять расположением элементов. Оба инструмента позволяют менять ориентацию и размеры блоков при изменении ширины экрана.

    ```CSS
    .container {
      display: flex;
      flex-wrap: wrap; /* Элементы переходят на новую строку при недостатке места */
    }
    ```

1. **Адаптивные изображения**: используйте изображения с `max-width: 100%; height: auto;`, чтобы они масштабировались с размером контейнера.

2. **Относительные единицы для шрифтов**: устанавливайте размеры шрифтов в `em` или `rem`, чтобы они адаптировались под размер экрана.

3. **CSS-свойство `clamp()`** позволяет задать адаптивный размер (например, шрифта), чтобы он оставался в заданном диапазоне.

    ```CSS
    .text {
      font-size: clamp(1rem, 2vw, 2rem);
    }
    ```

**Flex (Flexbox, Flexible Box Layout)** — это модель расположения элементов в CSS, которая позволяет легко строить адаптивные и гибкие интерфейсы.

<summary><b>Что такое Grid?</b></summary>

**CSS Grid** — это модель расположения элементов в CSS, которая позволяет строить **двумерные сетки** (по строкам и колонкам) для размещения элементов.

- определять строки (`grid-template-rows`) и колонки (`grid-template-columns`),

- легко управлять отступами (`gap`),

- делать адаптивные сетки с `fr`, `minmax()`, `auto-fit/auto-fill`.

В отличие от Flex (одномерной модели), **Grid управляет сразу и строками, и колонками**, что удобно для сложных макетов.

<details>
<summary><b>Для чего нужны препроцессоры?</b></summary>

**CSS-препроцессоры** (Sass, Less, Stylus) нужны, чтобы расширить возможности обычного CSS и сделать код более удобным для поддержки.

Что дают:

- **Переменные** (цвета, размеры, шрифты).

- **Вложенность селекторов** для читаемости.

- **Миксины** (функции со стилями).

- **Импорт файлов** и разделение кода на модули.

- **Условия и циклы** для генерации повторяющихся стилей.

Однако у многих из этих функций уже есть нативные аналоги в современном CSS:

|🔹 Возможность|CSS-препроцессор|Современный CSS|
|-|-|-|
|**Переменные**|Есть (`$color-primary: #333;`)|Есть (`--color-primary: #333;`)|
|**Вложенность селекторов**|Поддерживается (`.card { .title { … } }`)|Частично (через `:is()`, `:where()`, `&` — но не так гибко, пока только в черновиках спецификации Nesting)|
|**Миксины (функции со стилями)**|Есть (`@mixin button { ... }`)|Частично: кастомные свойства + `@layer`, `@property`, но прямого аналога нет|
|**Импорт и модули**|`@import`, `@use`, `@forward`|`@import` (устар.) и **CSS Modules** через сборщики|
|**Условия и циклы**|Есть (`@if`, `@for`, `@each`)|Нет (но можно эмулировать через `calc()`, `var()`, медиазапросы)|
|**Поддержка браузерами**|Требуется компиляция → CSS|Работает напрямую|
|**Экосистема**|Огромная, проверенная временем|Стандартизируется, всё больше возможностей без препроцессоров|
|**Гибкость и контроль**|Максимальная (можно генерировать тонны кода автоматически)|Чище, нативнее, меньше «магии»|

Cовременный CSS уже достаточно хорош и имеет много нативных возможностей, так что необходимость использовать препроцессоры постепенно уходит.

**Главный минус препроцессоров** — они требуют компиляции в нативный CSS.

<summary><b>Как реализовать темную/светлую тему в приложении?</b></summary>

Базовый подход состоит в определении глобальных css-переменных для цветов темы:

1. На корневой ноде (`html` или `body`) ставим флаг темы:
`class="dark"` или `data-theme="dark"`.

2. Все цвета/отступы/тени — через CSS custom properties (design tokens).

```CSS
:root {
  --primary-color: #ffffff;
  --text-color: #111827;
}

:root.dark {
  --primary-color: #020617;
  --text-color: #e5e7eb;
}

button {
  background: var(--primary-color);
  color: var(--text-color);
}
```

Таким образом нам не придется прописывать для каждого компонента вручную его цвет. Мы сможем использовать цветовые токены через `var()`, а их значения будут меняться в зависимости от наличия класса `.dark`

Переключение темы:

```JavaScript
function setTheme(theme) {
  document.documentElement.classList.toggle('dark', theme === 'dark');
  localStorage.setItem('theme', theme);
}
```

**Важно сохранять значение темы в localStorage, чтоб пользователю не приходилось каждый раз ее выставлять!**

Также можно учитывать системную тему через `prefers-color-scheme`:

```CSS
@media (prefers-color-scheme: dark) {
  :root {
    --primary-color: #020617;
    --text-color: #e5e7eb;
  }
}

```

Часто делают так:

```JavaScript
const saved = localStorage.getItem('theme');

if (saved) {
  // юзер явно выбрал тему → уважаем выбор
  document.documentElement.classList.toggle('dark', saved === 'dark');
} else {
  // иначе — смотреть на систему
  const prefersDark = window.matchMedia('(prefers-color-scheme: dark)').matches;
  document.documentElement.classList.toggle('dark', prefersDark);
}
```

Плюс можно слушать изменение системной темы:

```JavaScript
window.matchMedia('(prefers-color-scheme: dark)').addEventListener('change', e => {
  const theme = e.matches ? 'dark' : 'light';
  // применять, если юзер не зафиксировал свою
});
```
